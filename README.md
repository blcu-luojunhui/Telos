# Telos

> **记录生命，沉淀认知，建立秩序**  
> **Become who you are meant to be.**

**Telos** (τέλος) 是一个只属于我的私人 Agent。它记录我的生命，理解我的状态，承接我对世界的观察与感悟，沉淀我的认知，并帮助我在复杂、孤独而短暂的人生中，慢慢建立属于自己的秩序。

---

## 核心理念

### 这不是一个效率工具

Telos 不是待办清单，不是健康记录器，不是时间管理工具。

它是一个**面向我自己的长期系统**，用来：
- 记录生命的真实轨迹
- 理解自我的内在状态
- 观察世界的变化与规律
- 沉淀认知的结构与原则
- 建立属于自己的秩序

### 五层生命框架

Telos 理解的"生命"，不是单一维度的功能分类，而是一个从经历到秩序的五层结构：

#### 1. 自我 (Self)
关注"我是谁，我正在经历什么"
- 爱、孤独、勇敢、意志、坚强、恐惧、意义感、自我认同、成长

#### 2. 生活 (Life)
关注"我如何过好每一天"
- 健康、学习、工作、规划、习惯、节律、效率、决策

#### 3. 世界 (World)
关注"我如何观察外部世界"
- 对事件的观察、对人的理解、对关系的体会、对社会与现实的判断、对变化与规律的感悟

#### 4. 认知 (Cognition)
关注"我如何理解这些经历与观察"
- 判断、归因、反思、心智模型、信念更新、原则形成

#### 5. 秩序 (Order)
关注"我如何形成长期稳定的内在结构"
- 节律、原则、方法、决策框架、自律、长期方向、内在稳定感

这五层构成一条完整链路：

> **经历 → 感知 → 观察 → 认知 → 秩序**

---

## 为什么是 Self-Use

Telos 是一个 **Self-Use 产品**，当前阶段的唯一用户是我自己。

**1. 因为"我"本身就是最复杂、最长期、也最值得被理解的对象**
- 我的生命不是一个标准化需求集合，而是持续变化的整体
- 很多真正重要的问题无法被通用产品抽象成标准功能
- 只有 self-use，系统才能围绕"我是谁"持续演化

**2. 因为生命数据需要极高信任**
- 这里记录的不只是任务和健康，还包括孤独、恐惧、意志、观察、感悟与认知
- 这些内容比普通效率数据更私密、更脆弱
- 自己的数据自己掌控，系统才能真正承接真实的生命材料

**3. 因为这是一个长期生长的系统**
- 它的价值不在单次使用，而在长期积累
- 数据越完整，Agent 越能理解我的节律、模式、问题与成长轨迹
- 它不是一次性交付的工具，而是长期共生的系统

**4. 因为我自己就是最直接的验证场**
- 我能第一时间判断这个建议有没有价值
- 我能感受到这个系统是否真的懂我
- 我能快速迭代，而不被通用性约束拖慢

## 核心能力

### Phase 1: 健康维度 (已实现)

**记录能力**:
- 运动训练（类型、时长、距离、强度）
- 饮食记录（餐次、食物、热量）
- 身体指标（体重、体脂、睡眠）
- 状态心情（心情、精力、压力）
- 目标设定（减脂、增肌、备赛）

**查询能力**:
- 按时间范围查询运动/饮食/身体指标
- 多维度综合概览

**管理能力**:
- 编辑/删除记录
- 生成个性化训练计划

**决策能力** (开发中):
- 运动建议：基于历史和目标推荐训练
- 饮食建议：基于目标和当天摄入建议
- 恢复提醒：基于训练负荷提醒休息
- 目标追踪：目标进度分析和调整
- 趋势洞察：长期数据趋势分析

### Phase 2: 多维度扩展 (规划中)

- **学习维度**: 读书/课程/技能记录 + 复习提醒
- **工作维度**: 任务/会议/复盘记录 + 时间分析
- **情绪维度**: 情绪/能量记录 + 模式识别

### Phase 3: 跨维度智能 (规划中)

- 跨维度关联分析（运动与情绪、睡眠与效率）
- 整体平衡建议
- 月度生命复盘

- 行为模式学习
- 决策质量持续优化

## 开始使用

```bash
# 克隆项目
git clone https://github.com/yourusername/telos.git
cd telos

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的 API keys

# 启动服务
python -m src.main
```
