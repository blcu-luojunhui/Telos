# better me

**How to be a better me.**

为「我自己」打造一个长期可用的个人成长与生活 Agent：覆盖**成为更好的自己**所涉及的多个维度，以「我自己」的记录、记忆、规划作为上下文，提供即时建议，让「我自己」成为「我自己」。

## 项目目标

### 愿景：多维度「更好的我」

系统面向的是**全人**，而不是单一领域。涵盖的维度包括（并会随实践拓展）：

| 维度 | 说明 | 示例目标/记录 |
|------|------|----------------|
| **吃** | 饮食与营养 | 三餐记录、热量/营养素、饱腹感、与状态的关系 |
| **锻炼** | 运动与身体 | 跑步/篮球/力量等训练、负荷、恢复、身体指标 |
| **读书** | 阅读与输入 | 在读/读完的书、笔记、摘抄、与思考/输出的连接 |
| **思考** | 反思与心智 | 日记、复盘、决策记录、困惑与顿悟、心智模型 |
| **工作** | 效率与产出 | 要事、专注时间、项目进度、精力与工作匹配 |
| **社交** | 关系与连接 | 重要的人、见面/联系频率、社交能量与边界 |
| **睡眠** | 休息与恢复 | 睡眠时长与质量、作息规律、与情绪/表现的关系 |
| **情绪** | 心理与状态 | 心情、压力、动力、焦虑/平静，与各维度的交叉 |
| **学习** | 技能与成长 | 在学技能、练习记录、里程碑、与读书/工作联动 |
| **兴趣/创造** | 玩与创造 | 爱好、side project、放松方式、心流体验 |

一套**记忆、规划、目标、主观体验**的哲学。

### 核心能力（以「饮食 + 锻炼」为起点）

- **长期记忆**：记住我的训练记录（长跑、打篮球等）、饮食记录和身体指标变化；未来扩展至阅读轨迹、工作节奏、社交模式等。
- **周期性规划**：按月/按周生成训练+饮食计划，并根据执行情况自动调整；后续可做阅读计划、复盘节奏、工作 OKR 等。
- **即时建议**：根据当天的运动/饮食/状态，给出「今天/今晚/明天」的具体建议；可延伸到「今天适合深度工作还是沟通」「今晚要不要社交」等。
- **目标驱动**：支持设定目标（如「16 周后完成半马」），围绕目标生成周期化训练+饮食方案；同一套目标系统可承载读书目标、项目 deadline、习惯目标等。

## 设计哲学

系统在学什么、优化什么。

### 学习对象是「我这个人」，不是模板

普通 App 的范式是：背后有一堆训练/饮食模板，按你填的问卷参数从库里挑一个最合适的给你。这里的定位不同：**学习对象从「某个计划模板」升级成「你这一个人」**——包括你的训练、饮食、读书、思考、工作、社交、睡眠、情绪等如何相互影响。

- **输入**：训练负荷、饮食结构、生活压力/睡眠、时间（后续加入阅读量、工作负荷、社交频率等）。
- **输出**：你的主观状态（疲劳、心情、动力）+ 客观指标（体重、体脂、配速、心率等）。

也就是说，它要学的是：给你某种「周跑量 + 配速结构 + 饮食模式」组合后，一周、两周、四周后，你的身体和感受会怎样——好还是差。前者靠经验规则，后者用长期数据 + 记忆 + 简单统计/LLM，做**个体级别的自适应系统**。

### 优化的是多目标函数，而不是单一 KPI

目标不只是「瘦 10 斤/跑半马」这样的终点，而是显式定义一个**体验与结果兼顾的目标函数**，例如：

- **G₁**：体重/体脂朝目标区间移动。
- **G₂**：跑步表现（配速、完赛时间）。
- **G₃**：长期可持续性（不崩盘、不讨厌训练、不报复性反弹）。
- **G₄**：日常幸福感（心情、压力、睡眠感受）。

系统做的不是「最大化某一个」，而是在你设定的权重下做权衡，例如：半马备赛期提高 G₂ 的权重；比赛结束后提高 G₃、G₄ 的权重。未来甚至可以由 Agent 根据你的历史行为，反推你「嘴上说想减脂、实际更在意幸福感」的真实权重，并做温和引导。

### 终极形态：个人级「更好的我」Agent

- 你私有的 **better me 操作系统**——覆盖吃、锻炼、读书、思考、工作、社交、睡眠、情绪等多维度的**私人教练 + 生活顾问**。
- 为「你的一生」积累经验知识，跨年跨月的长程记忆，用来回答：
  - 哪几年你状态最好、那时候的训练–饮食–工作–社交–心情组合是怎样的；
  - 哪些阶段你反复崩盘或停滞、背后的模式是什么。
