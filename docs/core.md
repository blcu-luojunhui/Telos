## 项目核心

- **核心目标**：**better_me**（how to be a better me）—— 为「我自己」打造长期可用的个人成长与生活操作系统，覆盖吃、锻炼、读书、思考、工作、社交等多维度；

- **第一优先级：自我感受**
  - 系统优化的是「我这个人」的长期可持续的**体验与结果**，不是单一 KPI。主观感受（疲劳、心情、动力、压力、可持续性）与客观指标（体重、配速等）同等重要，且在发生冲突时，**优先不崩盘、不讨厌、可持续**，再追求数字目标。
  - 规划与记忆都围绕「什么组合让我状态好 / 什么组合让我崩」来积累和调参，而不是强行执行模板。

---

- **1. 目标组合与权重而非单一计划**
  - 允许同时存在多个目标（如「3 个月半马」「6 个月瘦 10 斤」），由 `WeeklyPlanner` 在生成计划时自动做权衡：
    - 关键训练日优先保证能量和恢复。
    - 在不影响训练质量的前提下，长期推进减脂等体型目标。

- **2. 深度个体化的长期记忆**
  - 这里通过 Memory Writer + 向量记忆，把「我过去的反应模式」长期积累起来：
    - 哪种跑量/配速组合我容易累垮。
    - 哪类饮食搭配我更有精神、更容易坚持。
  - 下次规划类似阶段时，会优先参考「自己的历史经验」，而不是公共模板。

- **3. 计划与现实的闭环调参**
  - 不只是「给一个 16 周计划」，而是每天/每周根据实际执行情况（训练完成度、体重变化、主观疲劳）动态调整：
    - 如果连续几周体重不降，就适度调整热量缺口或提高活动量。
    - 如果近期压力/疲劳过高，就自动降低训练强度或安排减量周。

- **4. 训练 + 饮食真正一体化**
  - 这里从架构上让它们共享同一个 `day_type` 和 `goal` 系统：
    - 关键课日 → 高碳水、足够热量。
    - 休息日 → 略低碳水、控制总热量。
  - 可以在同一个视图里看到「训练负荷–饮食–体重/体脂」之间的因果关系。

- **5. 针对「一个人」不断进化的教练风格**
  - 可以在对话与记忆中逐渐学习我的沟通偏好：
    - 更喜欢硬核数据还是温柔鼓励。
    - 更容易坚持的是严格打卡还是宽松建议。
  - 未来可以考虑让 agent 的「语言风格」「给建议的频率和方式」都随时间微调，真正像一个长期陪跑的教练。

- **6. 失败经验与长程复盘**
  - 系统会显式总结「失败周期」并写入记忆：既包括客观未达标（伤病、体重反弹），也包括**主观崩盘**（坚持不下去、情绪崩溃、报复性反弹）—— 原因可能是工作、饮食过于激进、社交被挤压等。在类似情境再次出现时，优先检索这些记忆，避免重复踩坑。
  - 跨年跨月的长程记忆，用来回答：哪几年状态最好、当时的训练–饮食–生活组合是什么；哪些阶段反复受伤或暴饮暴食、背后模式是什么。本质是在为「一生」积累经验知识图谱，而不是单次训练周期。
